# Workflow: Discussion → Plan → Detailed Instructions

## 1) Discussion
- Goal, scope, success criteria
- Constraints: time, budget, infra, dependencies
- Preferred repo/process (GitHub, mono/repo)

## 2) Plan
- High‑level phases with deliverables
- Risks/blockers
- Minimal timeline

## 3) Detailed instructions
- Step‑by‑step actions
- Role/task mapping
- Acceptance criteria and tests

## 4) Execution (optional)
- Track in issues/PRs
- Report status per `status.md`
